# Brewwery v0.9.0

Brewwery v0.9.0 is the Release Candidate for the first public Apple Silicon release.

## Focus

- Final packaging verification.
- Release Candidate documentation.
- Known limitations review.
- Security review checklist.
- Signing and notarization preparation.
- Final README/docs/screenshots polish.

## What Changed

- Version updated to `0.9.0`.
- App channel updated to `Release Candidate`.
- About, status bar, and bundle metadata target `Brewwery 0.9.0`.
- macOS signing entitlements are prepared for future Developer ID builds.
- Native Rust addon packaging scripts now explicitly build Apple Silicon and x64 targets before packaging.
- Known limitations are documented as release decisions.
- Feature freeze is documented before v1.0.

## Release Decision

Brewwery v0.9.0 remains unsigned and not notarized for Release Candidate testing.

The recommended v1.0 target is a signed and notarized Apple Silicon build if the Apple Developer account, Developer ID certificate, and notarization secrets are ready. If signing is not ready, keep v1.0 private/limited and document Gatekeeper instructions clearly rather than presenting it as a polished public install.

## Known Limitations

- Apple Silicon is the primary verified architecture.
- Intel and universal builds require dedicated validation. On this machine, `pnpm package:mac:x64` and `pnpm package:mac:universal` fail because the Rust `x86_64-apple-darwin` target is not installed.
- Local DMG creation can fail on some machines because of the APFS `hdiutil` issue.
- Cleanup and service actions do not yet use the live streaming progress panel.
- Open Terminal and package path actions remain hidden until they can be implemented safely.
- Builds are not auto-updated yet.

## Verification Commands

```bash
pnpm typecheck
pnpm lint
pnpm --filter @brewwery/desktop build
pnpm package:mac:dir
pnpm package:mac:zip
pnpm release:verify
```

Optional architecture validation:

```bash
pnpm package:mac:x64
pnpm package:mac:universal
```

## Security Summary

- No generic shell execution.
- No direct `ipcRenderer` calls from pages/components.
- Renderer uses the `brewweryApi` wrapper.
- Rust/Homebrew operations remain allowlisted.
- Streaming runner uses fixed argv arrays with `shell: false`.
- Custom Homebrew path is validated.
- Mutating actions require confirmation.
- Cleanup run requires preview first.
- `brew update` requires confirmation.
- Invalid search query input is rejected in renderer and Rust.
- No sudo.
- No telemetry.
- No cloud sync.
- No paid/pro/donation/support logic.
