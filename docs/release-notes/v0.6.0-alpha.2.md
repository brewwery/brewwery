# Brewwery v0.6.0-alpha.2

Private alpha polish release for macOS.

This release keeps the v0.6 private alpha feature set intact and focuses on small QA improvements for packaged app testing, issue reports, and release verification.

## What Changed

- Settings now includes a Copy diagnostics action for private alpha reports.
- About now shows the Private Alpha channel.
- Dashboard shows last-refresh state after manual refresh.
- Dashboard services preview prioritizes running services.
- History includes a failed-only filter for faster QA triage.
- Added `pnpm alpha:verify` for the local alpha verification sequence.
- Added `pnpm alpha:clean-install` for fresh local alpha testing.
- Added a private alpha test report template.

## Installation

1. Download `Brewwery-0.6.0-alpha.2-mac-arm64.dmg`.
2. Open the DMG.
3. Drag `Brewwery.app` into `/Applications`.
4. Launch Brewwery from `/Applications`.
5. Accept the unsigned app warning if you trust this private alpha build.

## Uninstall

Quit Brewwery first, then remove:

```bash
rm -rf "/Applications/Brewwery.app"
rm -rf "$HOME/Applications/Brewwery.app"
rm -rf "$HOME/Library/Application Support/Brewwery"
rm -f "$HOME/Library/Preferences/com.brewwery.app.plist"
rm -rf "$HOME/Library/Saved Application State/com.brewwery.app.savedState"
```

This does not remove Homebrew or installed Homebrew packages.

## Known Issues

- Builds are unsigned and not notarized.
- Apple Silicon is the primary tested architecture.
- Intel and universal builds are configured but may be untested.
- Local DMG creation can fail on some machines because of `hdiutil`; use `pnpm package:mac:dir` for local `.app` testing.
- Cleanup and service actions show final operation output rather than live progress.
- Tapped formula names containing `/` are rejected by the strict package-name validator.
- Package path and package-detail Open Terminal shortcuts are placeholders.
- No telemetry, crash reporting, cloud sync, paid features, or auto-updater are included.

## Security Notes

- No generic shell execution.
- No direct `ipcRenderer` usage in pages/components.
- Renderer operations use the typed `window.brewwery` API wrapper.
- Mutating actions require confirmation.
- Streaming runner uses fixed argv arrays with `shell: false`.
- No `sudo`.
- No automatic cleanup, install, uninstall, or upgrade.
